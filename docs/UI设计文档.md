# UI/UX 设计规范与体验架构 · ScriptForge

> **文档版本**：v1.1
>
> **面向对象**：前端开发、产品经理、视觉设计师
>
> **设计核心目标**：基于《PRD · ScriptForge》需求，构建兼具极简创作体验（C端）与强大工程及团队管理能力（B端）的企业级 AI 辅助创作工具。

## 1. 设计愿景与体验原则 (Design Principles)

- **克制与聚焦 (Focus & Immersion)**：编剧和作者是重度文字工作者。界面必须降低视觉噪音，采用极简的现代化设计（参考 Notion, Linear），推崇暗黑模式（Dark Mode），让内容（剧本）本身成为视觉界面的绝对主体。
- **渐进式暴露 (Progressive Disclosure)**：对 C 端个人作者隐藏复杂的批量任务与 API 字段，屏蔽底层的 YAML 代码概念，强调可视化卡片交互；对 B 端用户在管理控制台中提供强大的数据下钻、Webhook 配置与资源管理能力。
- **确定性与掌控感 (Control)**：AI 生成具有不确定性。UI 需要明确展示 AI 的处理状态（分段解析进度、流式渲染生成效果），支持多次版本历史溯源回退，并允许用户在可视化卡片与底层 YAML 代码之间无缝切换与人为手动重写。
- **剧本原子化呈现 (Atomic Narrative)**：在章节转换中落实 PRD 规定的“集 (Episode) -> 场景 (Scene) -> 节拍 (Beat)”规范，通过区块化的卡片设计，让底层复杂的剧本结构清晰可查、易于可控调序。

## 2. 角色化工作流设计 (Role-based UX Flows)

### 2.1 C 端创作者 (独立网文作者) —— “单线程心流模式”

- **核心路径**：导入文本（TXT/富文本） $\rightarrow$ 预览与确认章节切分 $\rightarrow$ 设置目标格式（短剧/长剧等） $\rightarrow$ AI 实时流式解析 $\rightarrow$ 进入工作台左右分屏对照打磨 $\rightarrow$ C 端订阅升级提示 $\rightarrow$ PDF/YAML 下载。
- **UX 重点**：降低“结构化数据”这一名词带来的认知负荷。默认展示“可视化编剧工作流”，将 YAML 配置转为“底层开发者视图”折叠呈现，强调极致流畅的文字续写与加工体验。

### 2.2 B 端企业用户 (量产型短剧公司) —— “看板与批处理模式”

- **核心路径**：大文本/多文件并发上传 $\rightarrow$ 任务队列管控 $\rightarrow$ 剧本阶段流转看板（解析中 / 待人工打磨 / 定稿） $\rightarrow$ 团队成员角色配置与分发 $\rightarrow$ 批量下载/ API 提取。
- **UX 重点**：大盘数据概览（本月 Token 调用量、并发成功率、处理时长统计）、批量操作管理界面。

### 2.3 B 端战略客户 (IP 孵化机构) —— “评估洞察模式” (P1)

- **核心路径**：单次导入全本小说 $\rightarrow$ 引擎抽取大纲与人物图谱 $\rightarrow$ AI生成「IP 改编可行性评估报告」 $\rightarrow$ 下载洞察报告。
- **UX 重点**：高呈现度的数据可视化（冲突密度分布折线图、出场人物关系网状图、视觉化潜力雷达图等），主打宏观决策分析。

## 3. 信息架构 (Information Architecture)

基于商业诉求规划，采用 **“一套底层，多角色视图”** 的侧边全局导航：

- **侧向全局导航 (Sidebar Global Nav)**
  - **📁 工作台 (Workbench)**：核心沉浸式创作区，管理当前正在编辑加工的单集/多章节项目。
  - **⚡️ 任务中心 (Task Center)**：批量上传与转化对列详情，展示后台异步处理进度（B端主视图）。
  - **📚 剧本库 (Assets)**：历史生成的剧本文件汇总、多格式归档与云端云盘存储管理。
  - **📊 IP 评估 (Insights) [阶段二 P1]**：面向版权与制片方，查阅系统化评分概览。
  - **🔌 API 与数据底座 (Dashboard)**：查看 Webhook 回调记录、API Key 凭证配置及服务订阅监控。
  - **⚙️ 设置与账单 (Settings)**：B/C 不同的计费套餐升购界面。

## 4. 核心界面交互设计需求 (Key Interface Modules)

### 4.1 导入、配置与预处理浮层 (Import & Onboarding)

由于需要处理规范化，必须设立清晰的引导：

- **上卷拖拽区**：大面积拖拽虚线框上传（.txt, .epub）；兼容直接粘贴小说内容的文本源。
- **预检步骤弹框 (Pre-flight Modal)**：
  - **章节映射确认**：扫描完文档后向用户展示切片章节清单（规则判定需 $\geq$ 3章），允许用户在提交云端算力前手动重组零碎段落。
  - **风格靶次定向**：提前定义改编形式（选择“竖屏短剧”、“悬疑长剧”），调整集尾悬念（Ending Hooks）分布密度。

### 4.2 核心主阵地：AI 剧本工作台 (The Workbench)

最复杂的旗舰体验页，采用 **响应式三栏分屏面板 (Split Panes)**，所有区块均支持自由拖拽伸缩宽度。

- **左侧面板：原著与上下文 (Context Pane)**
  - **原著对照 (Source)**：锁定光标自动在左侧高亮映射的原小说语句内容。
  - **人物档案 (Characters)**：聚合化的人物图谱抽出展示区，角色特征属性随时参考调用。
  - **故事大纲 (Outline)**：纵向的剧情节拍进度轴指示器，全局观览四幕结构。
  - _(注：此区域为只读参考信息辅侧面板)_

- **中央视野：可视化编辑器 (Visual Editor)**
  - 将冰冷的 YAML 模型转换为热烈的编剧互动体验。
  - **集级展示 (Episode Container)**：拥有暗色醒目的卡片横头，标识有「冷开场片头 (Cold Open)」输入区和「悬念扣子」区。
  - **场景卡片 (Scene Blocks)**：每一个 INT. / EXT., 以及地点与时间呈现为清晰的影视剧本排版。
  - **节拍交互列表 (Beat Rows)**：动作描述居左通栏；对白与内心独白根据登场角色搭配头像居中，高对比凸显潜台词。
  - **Inline Action (悬浮唤醒)**：选中某一句台词或节拍时，唤出“AI 增写剧烈情绪 / 一键润色为动作画面 / 场景情绪基调修改”快捷菜单工具链（支持底层 Schema 强力双向热更）。

- **右侧面板：AI 编剧助手与调试者视图 (Assistant & Inspector)**
  - **Tab 1: AI 协同窗**：如同对话副驾驶，响应用户的局部修改请求，并对特定复杂节点流式流出剧本推演优化版本。接受后即刻覆写左侧区域。
  - **Tab 2: 底层结构源 (YAML Code)**：为专业用户（技术型内容运营）提供的黑客帝国般的高亮代码编辑环境。如果手动输入 YAML 有格式错乱，JSON Schema 检测将即时产生编译下划红线错漏提示并尝试 AI 自动修正。
  - **Tab 3: 版本与时空追溯 (History / Revisions)**：由于修改极快且频繁，内置时光机体验表，提供最近至多 10 步历史快照供查看、Diff 对比与状态回退。

### 4.3 任务调度中心大盘 (面向 B 端团队开发)

- **全聚合任务列表 (Task Data Table)**：
  - 队列流水线显示：待分配排队 $\rightarrow$ 模型推理中 $\rightarrow$ Schema格式强验证校验中 $\rightarrow$ 完成提取。
- **故障反馈及一键重启**：大语言模型推理导致的幻觉失败任务将展开具体容错日志，点击重排队列，无门槛再推回系统池。

### 4.4 导出模式预处理视图 (Export Render Engine)

- 提供极像打印格式下的 PDF 所见即所得 (WYSIWYG) 实时富文本剧本排版视图。
- 多复选框并行分发：B端制作公司勾选配置后可以快速并行下载 YAML 模型、含专属水印的 PDF 文件以递交下游片场或 AI 工具流。

### 4.5 商业化引导策略层 (Monetization UI)

- **额度资源拦截网 (Paywall Check)**：当用户触发高级能力或超出该月份生成 Tokens/任务量阈值时，柔性化弹出磨砂玻璃订阅比对浮罩；明确标注“个人包月创作版”与“不限量接入API专业版/企业版”之差异。

## 5. UI/UX 动效与视觉表现 (Motion & Feedbacks)

- **流式感知 (Streaming Feedback)**：由于后台要处理数分钟的生成操作，工作台呈现骨架屏(Skeleton Loaders)以及渐进式的动态文本打字机效果 (Typewriter rendering) 填充，而非等待许久后卡顿式整页突变跳闪。
- **悬停内敛 (Hover Elegance)**：操作层按钮仅在焦点落在块级元素 (Hover active) 上以 Fade in 半透明渐变呼出，彻底清理不必要的繁杂视觉元件，守住心流边界。

## 6. 前端实现约束与技术生态契合度 (Implementation Strategy)

为确保该重资产型的轻量化产品高效构建与绝佳交互性能表现，强烈建议前端视觉工程结合以下现代技术栈规范标准落实开发：

- **界面底座与样式模型**：高度推荐采用 **Next.js (App Router)** 全栈范式协同 **Tailwind CSS**。借助 Tailwind 实用优先类名控制以最快效率编写高度复杂的紧凑视图面板，且天然适配完美的暗黑/明亮工作域的主题转换。
- **高性能无访问壁垒组件 (a11y)**：
  - 基于 **shadcn/ui** 以及 **Radix UI** 拼装系统层级组件。它们提供极其优秀的无渲染 Headless 可定制能力，保证体验上限的同时避开笨重传统框架（如 Ant Design 或 Element UI）的模板化桎梏。
- **复杂视图结构的状态流动管理**：
  - 由于具备四块核心面板高度联动的逻辑（大纲纲要 $\leftrightarrow$ 场景组件图卡 $\leftrightarrow$ YAML源码树 $\leftrightarrow$ AI 副驾响应），强烈部署 **Zustand** 进行全局颗粒态数据状态池的映射代理追踪。利用 `react-split` 等实现响应式高流畅拖拽窗栏。
- **大模型响应与流式推演双向互融 (Streaming + LLM Integration)**：
  - 在大面积调用流式反馈交互下，建议高度依赖开源范例文库优异的 **Vercel AI SDK** 解决与 AI Provider 层的数据接口绑定。深度应用它的 Streaming 工具链支持、`useChat` 编排、以及 Function Calling (Tool calls) 机制构建 AI Assistant 侧边栏和实时修改响应窗，将复杂的延时请求响应抹平如丝般通畅顺滑。
