## 1. 总体设计原则

| 原则 | 说明 |
|---|---|
| **RESTful 优先** | 资源以名词复数命名；标准动词 GET/POST/PATCH/DELETE |
| **JSON 为主，YAML 为辅** | 所有 API 请求响应 JSON；剧本内容字段可携带 YAML 文本或解析后的 JSON 对象，由 `format` 参数控制 |
| **异步任务模型** | 转换操作不阻塞 HTTP；返回 `task_id` + 进度查询/SSE/Webhook 三种获知结果 |
| **幂等性** | 写操作支持 `Idempotency-Key` 请求头 |
| **可观测** | 所有响应携带 `X-Request-Id` 便于排障 |
| **OpenAPI 3.1** | FastAPI 自动生成 `/openapi.json`，对外文档站基于此构建 |
| **双端共用** | 同一套接口同时服务 Web 前端（Cookie/JWT）与 B 端（API Key） |

## 2. 通用约定

### 2.1 基础信息

| 项 | 值 |
|---|---|
| **Base URL（生产）** | `https://api.scriptforge.cn/v1` |
| **Base URL（沙箱）** | `https://api-sandbox.scriptforge.cn/v1` |
| **Content-Type** | `application/json; charset=utf-8` |
| **时间格式** | ISO 8601，含时区，如 `2026-06-05T10:00:00+08:00` |
| **字符编码** | UTF-8 |
| **ID 格式** | ULID（26 位字符串），以资源类型短码前缀，如 `usr_01H...`、`proj_01H...` |

### 2.2 标准请求头

| Header | 说明 |
|---|---|
| `Authorization` | `Bearer <token>` 或 `ApiKey <key>` |
| `X-Request-Id` | 客户端可选传入，便于排障；后端无则生成 |
| `Idempotency-Key` | 写操作幂等键（UUID v4），有效期 24h |
| `Accept-Language` | `zh-CN` / `en-US` |

### 2.3 统一响应结构

**成功**：直接返回资源对象或集合，HTTP 2xx。

```json
{
  "id": "scrpt_01H...",
  "title": "斗破苍穹·改编",
  "created_at": "2026-06-05T10:00:00+08:00"
}
```

**集合响应（分页）**：

```json
{
  "data": [ { "...": "..." } ],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 153,
    "has_next": true,
    "next_cursor": "eyJpZCI6Li4ufQ=="
  }
}
```

**错误**：HTTP 4xx/5xx，统一结构：

```json
{
  "error": {
    "code": "INVALID_CHAPTER_COUNT",
    "message": "至少需要 3 个章节才能进行转换。",
    "details": { "received": 1, "min_required": 3 },
    "request_id": "req_01H..."
  }
}
```

### 2.4 分页

- **页码分页**：`?page=1&page_size=20`（默认 20，上限 100）
- **游标分页**（推荐高频列表）：`?cursor=xxx&page_size=20`

### 2.5 字段过滤

支持 `?fields=id,title,status` 仅返回指定字段，降低带宽。

## 3. 鉴权与会话

### 3.1 鉴权方式

| 场景 | 方式 | 适用 |
|---|---|---|
| Web 用户 | Supabase Auth JWT（`Authorization: Bearer <jwt>`） | 前端用户操作 |
| B 端服务 | API Key（`Authorization: ApiKey sk_live_...`） | 服务器对服务器 |
| Webhook 回调 | HMAC-SHA256 签名（请求头 `X-ScriptForge-Signature`） | 客户系统验证回调真实性 |

### 3.2 API Key 设计

- 前缀区分环境：`sk_live_*`、`sk_test_*`
- 仅创建时返回明文，后续仅展示后 4 位
- 支持 scope 限制：`scripts:read`、`scripts:write`、`tasks:*`
- 支持 IP 白名单与速率限制

### 3.3 用户接口

| 方法 | 路径 | 描述 |
|---|---|---|
| POST | `/auth/signup` | 邮箱注册（验证码） |
| POST | `/auth/login` | 邮箱密码登录 |
| POST | `/auth/oauth/{provider}` | 微信 / GitHub OAuth（M4+） |
| POST | `/auth/logout` | 注销 |
| POST | `/auth/refresh` | 刷新 token |
| GET | `/auth/me` | 当前用户信息 |
| POST | `/auth/api-keys` | 创建 API Key |
| GET | `/auth/api-keys` | 列出 API Key（仅展示后 4 位） |
| DELETE | `/auth/api-keys/{key_id}` | 撤销 API Key |

## 4. 资源模型

```
User (用户)
 └─ Project (项目 = 一本小说的改编工作区)
     ├─ Chapter[] (章节，原始文本)
     ├─ Task[] (转换任务)
     └─ Script[] (剧本版本，YAML 内容)
         └─ Episode[] → Scene[] → Beat[]
```

| 资源 | ID 前缀 | 关键字段 |
|---|---|---|
| User | `usr_` | id, email, plan, quota_used, quota_total |
| Project | `proj_` | id, user_id, title, source_novel, source_author, created_at |
| Chapter | `chap_` | id, project_id, index, title, text, token_count |
| Task | `task_` | id, project_id, type, status, progress, error |
| Script | `scrpt_` | id, project_id, version, yaml_content, schema_version |
| Webhook | `whk_` | id, user_id, url, events, secret |

### 6.3 文件上传（章节导入）

#### `POST /projects/{project_id}/chapters:upload`

**Content-Type**：`multipart/form-data`

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | binary | 是 | TXT/EPUB，最大 500KB |
| `auto_detect_chapters` | bool | 否 | 默认 true |
| `encoding` | string | 否 | 不传则自动检测（UTF-8 / GBK） |

**响应 200**：

```json
{
  "uploaded_bytes": 384210,
  "encoding_detected": "UTF-8",
  "chapters": [
    { "id": "chap_01H...", "index": 1, "title": "第一章 陨落的天才", "token_count": 2840 },
    { "id": "chap_01H...", "index": 2, "title": "第二章 斗气大陆", "token_count": 3120 }
  ],
  "warnings": []
}
```

**错误**：

- `400 FILE_TOO_LARGE`
- `400 UNSUPPORTED_ENCODING`
- `422 NO_CHAPTERS_DETECTED`

### 6.5 提交转换任务

#### `POST /projects/{project_id}/tasks`

```json
{
  "type": "novel_to_script",
  "chapter_ids": ["chap_01H...A", "chap_01H...B", "chap_01H...C"],
  "options": {
    "schema_version": "1.0",
    "act_structure": "four_act",
    "include_monologue_visualization": true,
    "tone_preference": ["dramatic", "romantic"],
    "model_preference": "auto"
  },
  "webhook_id": "whk_01H..."
}
```

**响应 202 Accepted**：

```json
{
  "id": "task_01H...",
  "project_id": "proj_01H...",
  "type": "novel_to_script",
  "status": "queued",
  "progress": 0,
  "created_at": "2026-06-05T10:00:00+08:00",
  "estimated_seconds": 180,
  "events_url": "/v1/tasks/task_01H.../events"
}
```

**校验规则**：

- `chapter_ids` 至少 3 个，至多 50 个
- 单章节 token 数不超过 30k

**错误**：

- `400 INVALID_CHAPTER_COUNT`
- `402 QUOTA_EXCEEDED`
- `429 RATE_LIMITED`

### 6.8 剧本局部更新（编辑器）

#### `PATCH /scripts/{script_id}`

使用 [JSON Patch (RFC 6902)](https://datatracker.ietf.org/doc/html/rfc6902) 表达细粒度修改，避免整体覆盖。

```json
{
  "patches": [
    {
      "op": "replace",
      "path": "/content/episodes/0/scenes/2/beats/4/dialogue/line",
      "value": "我命由我不由天！"
    },
    {
      "op": "add",
      "path": "/content/episodes/0/scenes/2/beats/-",
      "value": {
        "beat_id": "bt_new_001",
        "type": "action",
        "action": { "description": "萧炎转身离去，背影坚定。" }
      }
    }
  ],
  "expected_version": "1.0.0"
}
```

**响应 200**：

```json
{
  "id": "scrpt_01H...",
  "version": "1.0.1",
  "schema_valid": true,
  "applied": 2,
  "warnings": []
}
```

**冲突**：

- `409 VERSION_CONFLICT`：`expected_version` 与服务端不符，返回最新版本号
- `422 SCHEMA_VALIDATION_FAILED`：补丁导致 Schema 校验失败，原文不变

## 7. 实时进度（SSE）

### 7.1 端点

```
GET /tasks/{task_id}/events
Accept: text/event-stream
Authorization: Bearer ...
```

### 7.2 事件类型

```
event: progress
data: {"progress": 42, "step": "transform_chapter:2"}

event: chapter_completed
data: {"chapter_id": "chap_01H...", "scenes": 14}

event: log
data: {"level": "info", "message": "Schema 校验通过"}

event: completed
data: {"script_id": "scrpt_01H...", "metrics": {"total_scenes": 38}}

event: failed
data: {"code": "LLM_TIMEOUT", "message": "模型响应超时"}
```

### 7.3 客户端实现要点

- 连接断开后自动 `Last-Event-Id` 续传
- 心跳 `event: ping` 每 15s 发送
- 任务完成或失败后服务端主动关闭流

## 8. Webhook 回调

### 8.1 订阅

#### `POST /webhooks`

```json
{
  "url": "https://example.com/scriptforge/callback",
  "events": ["task.completed", "task.failed", "task.partial_succeeded"],
  "secret": "whsec_xxx"
}
```

### 8.2 回调请求

ScriptForge → 客户端：

```http
POST /scriptforge/callback HTTP/1.1
Content-Type: application/json
X-ScriptForge-Event: task.completed
X-ScriptForge-Delivery: dlv_01H...
X-ScriptForge-Signature: t=1717574400,v1=hex(hmac_sha256(secret, "{ts}.{body}"))

{
  "event": "task.completed",
  "ts": "2026-06-05T10:05:00+08:00",
  "data": {
    "task_id": "task_01H...",
    "project_id": "proj_01H...",
    "script_id": "scrpt_01H...",
    "metrics": { "total_scenes": 38 }
  }
}
```

### 8.3 校验示例

```python
import hmac, hashlib
sig_header = headers["X-ScriptForge-Signature"]
ts, v1 = parse(sig_header)
expected = hmac.new(secret.encode(), f"{ts}.{body}".encode(), hashlib.sha256).hexdigest()
assert hmac.compare_digest(expected, v1)
```

### 8.4 重试策略

- 客户端必须在 5 秒内返回 2xx
- 失败后指数退避重试：1m、5m、30m、2h、12h，共 5 次
- 5 次仍失败标记 Webhook 不健康，连续 24h 不健康则自动停用

## 9. 错误码表

| HTTP | code | 含义 |
|---|---|---|
| 400 | `INVALID_REQUEST` | 参数格式错误 |
| 400 | `INVALID_CHAPTER_COUNT` | 章节数不足或超限 |
| 400 | `FILE_TOO_LARGE` | 文件超过 500KB |
| 400 | `UNSUPPORTED_ENCODING` | 编码无法识别 |
| 401 | `UNAUTHENTICATED` | 缺少或无效鉴权 |
| 403 | `FORBIDDEN` | 权限不足 |
| 404 | `NOT_FOUND` | 资源不存在 |
| 409 | `VERSION_CONFLICT` | 乐观锁冲突 |
| 409 | `IDEMPOTENCY_KEY_CONFLICT` | 幂等键复用但请求体不同 |
| 422 | `SCHEMA_VALIDATION_FAILED` | YAML Schema 校验失败 |
| 422 | `NO_CHAPTERS_DETECTED` | 未识别到章节 |
| 402 | `QUOTA_EXCEEDED` | 套餐额度用完 |
| 429 | `RATE_LIMITED` | 触发限流 |
| 451 | `CONTENT_VIOLATION` | 内容审核未通过 |
| 500 | `INTERNAL_ERROR` | 服务内部错误 |
| 502 | `LLM_UPSTREAM_ERROR` | 大模型上游错误 |
| 503 | `LLM_TIMEOUT` | 大模型超时 |

## 10. 限流策略

| 维度 | 默认限制 | Header |
|---|---|---|
| 匿名 IP | 30 req/min | `X-RateLimit-*` |
| 免费用户 | 60 req/min | 同上 |
| 创作版 | 300 req/min | 同上 |
| API Key（团队） | 1000 req/min + 50 并发任务 | 同上 |
| API Key（企业） | 5000 req/min + 300 并发任务 | 同上 |

响应头：

```
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 287
X-RateLimit-Reset: 1717574460
```

触发限流：HTTP 429，`Retry-After` 标明等待秒数。

## 11. 版本与兼容性

- **URL 版本**：`/v1`，重大不兼容变更升级到 `/v2`
- **Schema 版本**：剧本 YAML 内 `metadata.schema_version`，与 API 版本解耦

- **新增字段**：客户端必须容忍未知字段（forward compatibility）

## 12. SDK 设计示意

### 12.1 Python SDK

```python
from scriptforge import ScriptForge

sf = ScriptForge(api_key="sk_live_xxx")

project = sf.projects.create(
    title="斗破苍穹·改编",
    source_novel="斗破苍穹",
    format={"type": "micro_drama_vertical", "total_episodes": 80},
)

with open("novel.txt", "rb") as f:
    chapters = sf.chapters.upload(project_id=project.id, file=f)

task = sf.tasks.create(
    project_id=project.id,
    chapter_ids=[c.id for c in chapters[:5]],
)

for event in task.stream():
    if event.type == "completed":
        script = sf.scripts.get(event.data["script_id"], format="yaml")
        print(script)
        break
```

### 12.2 Node SDK

```ts
import { ScriptForge } from "@scriptforge/sdk";

const sf = new ScriptForge({ apiKey: process.env.SF_KEY });

const project = await sf.projects.create({ title: "斗破苍穹·改编", /* ... */ });
const chapters = await sf.chapters.upload(project.id, file);
const task = await sf.tasks.create({ projectId: project.id, chapterIds: chapters.map(c => c.id) });

for await (const ev of task.stream()) {
  if (ev.type === "completed") {
    const yaml = await sf.scripts.export(ev.data.script_id, "yaml");

  }
}
```
